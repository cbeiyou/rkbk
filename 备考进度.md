## 学习笔记

### 第六章 项目管理概论 - 核心知识点

**1. 项目三要素**
- 临时性：明确的开始和结束时间
- 独特性：创造独特的产品、服务或成果
- 渐进明细：逐步细化

**2. 项目管理三大技能**
- 项目管理技术
- 战略和商务
- 领导力

**3. 组织结构类型**
| 类型 | 项目经理权限 | 资源可用性 |
|------|--------------|------------|
| 职能型 | 极少或无 | 极少或无 |
| 弱矩阵 | 低 | 低 |
| 平衡矩阵 | 低到中 | 低到中 |
| 强矩阵 | 中到高 | 中到高 |
| 项目型 | 高到几乎全部 | 高到几乎全部 |

**4. PMO三种类型**
- 支持型：提供支持，控制程度低
- 控制型：提供支持+要求合规
- 指令型：直接管理控制项目，控制程度最高

**5. 项目生命周期特征**
- 初始阶段：成本/人员投入低，不确定性最高
- 中间阶段：成本/人员投入达到最高
- 收尾阶段：变更代价最高

**6. 五大过程组**
- 启动：制定项目章程、识别干系人
- 规划：制定项目管理计划
- 执行：花费预算最多
- 监控：控制变更、纠正措施
- 收尾：结束项目或阶段

**7. 项目组合 vs 项目集 vs 项目**
- 项目组合：做正确的事（优先级排序，战略一致）
- 项目集：正确地做事（协调依赖关系）
- 项目：实现具体目标

### 第七章 立项管理 - 核心知识点

**1. 立项流程**
```
项目建议书 → 可行性研究 → 项目评估 → 项目决策
   (可省略)    (初步+详细)   (第三方)
```

**2. 项目建议书**
- 编写方：项目建设单位
- 核心内容：必要性、市场预测、预期成果市场预测、建设必需条件
- 可省略（小项目可与可行性分析合并）

**3. 可行性研究类型**

| 类型 | 关键词 | 可否省略 |
|------|--------|----------|
| 机会研究 | 激发投资者兴趣、寻找投资机会 | 可省略 |
| 初步可行性研究 | 初步评估、粗略估算 | 可省略 |
| 详细可行性研究 | 深入调查研究、费时费力 | **不可省略** |

**4. 可行性分析内容**

| 类型 | 关键词/方法 |
|------|-------------|
| 技术可行性 | 开发人员、技术方案比较、人力资源、设备选型 |
| 经济可行性 | 投资、成本、收益、**敏感性分析** |
| 运行环境可行性 | 用户管理体制、人员素质、数据资源积累 |
| 法律可行性 | 合同责任、知识产权 |

**5. 项目评估**
- 执行方：**第三方**（国家、银行或有关机构）
- 目的：审查**可靠性、真实性、客观性**
- 不能由建设单位执行！

**6. 支出分类**
- 一次性支出：设备购置费、差旅费、培训费
- 非一次性支出：软硬件租金、人员工资福利、水电费

**7. 经营成本 vs 非经营成本**
- 经营成本：研发成本、行政管理费、销售与分销费用
- 非经营成本：财务费用、折旧、摊销

**8. 有无比较法 vs 前后比较法**
- **有无比较法**：更准确！考虑"不上项目时的变化趋势"
- 前后比较法：简单对比，可能夸大或低估效益

项目整合管理 - 核心知识点

1. 项目整合管理的过程定义和作用
   1. 制定项目章程，定义：编写一份正式批准项目并授权项目经理在项目活动中使用组织资源的文件。作用：明确项目与组织战略之间的直接练习。确立项目的正式地位。展示组织对项目的承诺。开展一次或在项目的预义时开展。
   2. 制定项目管理计划：定义：定义准备和协调项目计划的所有组成部分，把他们整合为一份综合项目管理计划的过程。作用：生成一份综合文件，确定所有项目工作的基础和执行方式。
   3. 指导与管理项目工作：定义：执行工作，实施已批准变更的过程。作用：对项目工作和可交付成果开展综合管理，提高项目成功的可能性。在整个项目期间开展。
   4. 管理项目知识：定义：使用现有知识，生成新的知识，实现项目目标，帮助组织学习的过程。作用：利用知识来创造和改进项目成果。使当前项目创造的知识可用于支持组织运营和未来的项目或阶段。在整个项目期间开展。
   5. 监控项目工作：定义：跟踪审查报告整体项目进展。实现项目管理计划中绩效目标的过程。作用：让干系人了解当前项目的当前状态并认可处理绩效问题采取的行动。采取成本和进度预测，让干系人了解项目情况。在整个项目期间开展。
   6. 实施整体变更控制：定义：审查变更请求。批准变更管理，对可交付成果，项目文件，项目管理计划的变更。
   7. 结束项目或阶段。定义：终结项目、阶段、合同的所有活动；作用：存档项目或阶段信息，完成计划的工作。释放资源。只开展一次，或者在阶段结束点开展。
2. 项目整合管理由项目经理负责，不允许授权或转移。
3. 最好在项目章程制定时任命项目经理。项目章程可以发起人编制，也可以由项目经理和发起机构合作。
4. 变更请求：纠正：为了使得项目工作绩效和项目管理计划一致而进行的活动、预防：确保项目工作未来的绩效符合项目管理计划而进行的活动、缺陷补救：修正不一致产品或者组件的活动。更新：对项目文件或者计划进行变更，反应修改、增加的意见或者内容。
5. 问题的日志：类型、提出者和提出时间、问题描述、优先级、负责人、目标解决日期、状态、最终解决情况
6. 管理项目知识：输入：项目管理计划，项目文件，可交付成果，事业环境因素，组织过程资产。技术：专业判断、知识管理、信息管理、人际关系和团队技能。输出：经验校验登记册，项目管理计划更新。组织过程资产更新。
7. 项目范围管理：规划范围管理、收集需求、定义范围、创建wbs、确认范围、控制范围
8. 管理质量的工具：数据收集、数据分析、数据表现、审计、面向X的设计、问题解决、质量改进方法
9. 控制质量的目的：评估绩效，对可交付成果的质量进行内部验收。判断可交付成功是否达到客户质量要求。所以输入有：项目管理计划、项目文件（经验教训登记册、质量测量指标、测试和评估文件）、批准的变更请求
10. 管理质量：=质量保证+过程改进。所以要先做数据收集、数据分析、数据决策、数据表现；然后进行审计、面向X的设计、问题解决、质量改进方法；输出：数量报告、变更请求、测试和评估文件、项目管理计划更新、项目文件更新
11. 项目质量管理三过程区分：
    - 规划质量管理：制定标准、确定测量指标 → 输出：质量管理计划、质量测量指标
    - 管理质量：审计过程、过程改进 → 输出：质量报告、测试与评估文件、变更请求
    - 控制质量：测试、检查、核实 → 输出：质量控制测量结果、核实的可交付成果、工作绩效信息
12. 质量成本分类：
    - 一致性成本：预防成本（培训、流程文档化）+ 评价成本（测试、检查）
    - 非一致性成本：内部失败成本（返工、报废）+ 外部失败成本（保修、业务流失）
    - 关键：预防成本 < 纠正成本；内部失败成本也是质量成本！
13. 质量管理工具对比：
    - 流程图：分析过程步骤，找出问题环节
    - 因果图（鱼骨图）：识别根本原因
    - 控制图：判断过程是否稳定/可控，反映可允许的最大值和最小值
    - 直方图：描述集中趋势、分散程度、分布形状
    - 散点图：展示两个变量之间的关系
    - 帕累托图：识别主要问题（80/20原则）
    - 亲和图：对缺陷成因分类，展示最应关注的领域
    - 矩阵图：展示因素间强弱关系
14. 核对单 vs 核查表：
    - 核对单：以结构化方式管理控制质量活动（步骤清单）
    - 核查表：收集关于潜在质量问题的有用数据（数据收集）
15. 质量审计目标：
    - 识别良好及最佳实践
    - 识别违规做法、差距及不足
    - 分享良好实践
    - 协助改进过程执行
    - 对经验教训积累做出贡献
    - 注意：确定过程是否可控是控制图的作用，不是审计目标
16. 项目合同是项目质量管理的主要依据（质量要求反映在合同中） 



### 项目进度管理

+ 规划进度管理： 输出 进度管理计划

+ 定义活动：输出 活动清单、活动属性、里程碑清单、变更请求、项目管理计划更新（进度基准、成本基准）

+ 排列活动顺序：输出：项目进度网络图、项目文件更新

+ 估算活动持续时间：输出：持续时间估算、估算依据，项目文件更新。工具和技术：专家判断、类比估算-粗略快、参数估算-公式算、可准确、三点估算、自下而上估算、数据分析（备选方案分析和储备分析）、决策和会议

+ 制定进度计划：输出：进度基准、项目进度计划、进度数据、项目日历、变更请求、项目管理计划更新、项目文件更新。工具和技术：进度网络分析、关键路径法、资源优化、数据分析（假设情景分析、模拟）提前量和滞后量、进度压缩、项目管理信息系统、敏捷发布规划

+ 控制进度

1. 项目进度管理计划的内容：进度模型制定、准确度、计量单位、项目进度模型维护、控制临界值、绩效测量正规化、报告格式、过程描述、工作分解结构、进度计划的发布和迭代
2. 制定进度计划的关键步骤：
   1. 定义项目里程碑，识别活动并排列活动顺序、估算持续时间、确定活动开始和完成日期
   2. 分配至各个活动的项目人员审查被分配的活动
   3. 项目人员确认开始和结束日期与资源日历和其他项目或任务没有冲突，确认计划日期的有效性
   4. 分析进度计划，确认是否存在冲突，在批准进度计划并将其作为基准之前确认是否需要进行资源平衡，确保进度计划在整个项目期间一直切实有效。
3. 资源优化技术：
   1. 资源平衡：针对关键资源，会导致关键路径变化，一般是延长
   2. 资源平滑：不概念关键路径，针对非关键资源，完工日期不会延迟。
4. 进度压缩：
   1. 赶工：最小成本最大限度压缩进度。加班加人。导致风险和成本增加。
   2. 快速跟进：串行活动改并行。可能造成返工。可以缩短工期。
